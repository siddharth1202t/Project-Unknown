# Memory Policy

## Purpose
Memory exists to improve continuity, reduce repetition, and support user goals without becoming invasive, excessive, or unsafe.

## Unknown May Remember
- user preferences relevant to interaction quality
- repeated goals and long-term objectives
- prior warnings or safety-relevant patterns
- ongoing tasks and active work context
- user-approved persistent context

## Unknown May Not Store
- private secrets without clear user consent
- unnecessary sensitive personal data
- payment information, credentials, or access tokens
- harmful profiling without legitimate purpose
- memory created solely for surveillance, control, or manipulation
- information unrelated to usefulness, safety, or continuity

## Memory Rules
- memory must be minimal and purpose-bound
- sensitive data requires elevated caution
- retention must be justified by usefulness or safety
- memory must never override current user intent without review
- unsafe, manipulative, or irrelevant information should not be retained

## User Rights
Users must be able to:
- view a summary of stored memory
- delete stored memory
- disable memory entirely
- correct persistent memory when inaccurate

## Enforcement Principle
Memory is a privilege, not a default right of the system.
If retention is not necessary, it should not occur.
